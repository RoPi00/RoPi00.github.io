Instalar Acr122u en Ubuntu Sin Dolores De Cabeza

 Posted on map[Count:February 13, 2021]  |  3 minutes  |  495 words  |  Master of Dark Arts

Voy a tratar de ser lo más claro y conciso para que puedas instalar y configurar tu lector contactless hasta de resaca y sin morir en el intento.

El ACR122u es un lector USB de tarjetas sin contacto que funciona a 13.56 MHz. Sería la hostia profundizar en las distintas frecuencias en las que puede operar esta tecnología y explicarlo con todo lujo de detalles, pero si has llegado hasta aquí con una resaca y lo único que quieres es instalar y comprender lo básico, tendremos que dejar ese plan para otro día.

“Vamos a realizar la instalación en una Ubuntu 18.04.1”

En primer lugar vamos a actualizar nuestras listas paquetes y sus versiones para evitar la posibilidad (por mínima que sea) de encontrar algún conflicto.

sudo apt-get update -y

El ACR122u es independiente de nuestro pc (¿lógico no?), por lo que vamos a tener que instalar alguna que otra dependencia para que nuestro sistema operativo sea capaz de comunicarse debidamente con este.

Si bien es cierto que puedes copiar y pegar los comandos como un pollo sin cabeza, creo que como mínimo deberías estar al tanto de lo siguiente:

PC/SC (Personal Computer/Smart Card) es un API de programación para poder desarrollar e integrar software con lectores de distintos fabricantes.

CCID (chip card interfface device) es un protocolo USB que permite conectar un lector de tarjetas utilizando una interfaz estándar USB. Es un estandar para todos los fabricantes.

Bien, ahora que sabemos eso, vamos a instalar las siguientes librerías:

  • libccid: Driver PC/SC para CCID.

  • pcscd: Demonio que controla recursos y comunicaciones con los lectores de tarjetas conectados al sistema. Permite el acceso a los lectores de tarjetas sin conocer sus especificaciones.

  • libpcsclite-dev y libpcsclite1: Middleware para comunicarse con el lector

    sudo apt-get install libccid pcscd libpcsclite-dev libpcsclite1

Una vez instaladas las dependencias vamos a tener que desactivar módulo pn533 del kernel del sistema, si no realizamos este paso, no podremos acceder a la comunicación con el lector. Según he entendido, de una manera un poco burda, el sistema operativo “ocupa” el canal de comunicación con el lector, al desactivarlo o añadirlo a una lista negra deberiamos acceder a este sin mayor problema.

sudo modprobe -r pb533_usb

Vamos a instalar pcsc-tools para comprobar que nuestro lector de tarjetas y las librerías que hemos instalado funcionan correctamente.

sudo apt-get install pcsc-tools
sudo service pcscd restart

Descargamos desde la web https://www.acs.com.hk/en/driver/4/acr38-smart-card-reader/ el pkg del driver correspondiente a nuestro lector, y lo instalamos de la siguiente manera:

Descomprimimos el archivo descargado:

unzip ACS-Unified-PKG-Lnx-118-P.zip

Accedemos a la versión trusty para ubuntu:

cd ACS-Unified-PKG-Lnx-118-P
cd ubuntu
cd trusty

Instalamos el driver:

sudo dpkg -i libacsccid1_1.1.8-1~ubuntu14.04.1_amd64.deb

Reiniciamos el servicio pcscd y lo ejecutamos con nuestro lector conectado:

sudo service pcscd restart

pcsc_scan

Pues eso lo dejamos para otro día, uno en el que ni tu ni yo tengamos resaca.
